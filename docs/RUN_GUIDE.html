3. Run it locally

  1. Environment file — already done. .env exists at the repo root with a 256-bit JWT_SECRET, ADMIN_BA_IDS=TMBA-FOUNDER-KEVIN, and direct persistence values for MongoDB, Neo4j, ChromaDB, and the GPU embedder.
  2. Free port 7700. Another process currently holds it, and all three front-ends are hard-wired to call the API there, so v2’s server must own 7700. Identify and stop the occupant (or decide it is the one to keep). 
    Get-NetTCPConnection -LocalPort 7700 -State Listen |
  ForEach-Object { Get-Process -Id $_.OwningProcess } | Format-List Id,Path
  3. Install dependencies (likely already installed): 
    cd D:\momentum-creation-system-v2
pnpm install
  4. Build the shared package once (so imports resolve): 
    pnpm build:shared
  5. Start everything (server + all three clients in parallel): 
    pnpm dev:all
    Or run separately: pnpm dev:server, pnpm dev:com, pnpm dev:team, pnpm dev:admin.
  6. Open in the browser:
    • http://localhost:7701 — prospect site (public, no login) — fastest thing to see.
    • http://localhost:7702 — BA cockpit (needs login).
    • http://localhost:7703 — admin (needs login).
  7. Seed data so logins work (otherwise the database is empty). From the repo root: 
    pnpm --filter @momentum/server seed:founders
pnpm --filter @momentum/server setup:founder-access
pnpm --filter @momentum/server seed:codes
pnpm --filter @momentum/server seed:webinar-events
pnpm --filter @momentum/server seed:orientation-sessions
pnpm --filter @momentum/server bootstrap:ivory
    Confirm which of these are required for a working login before running blind — founders + founder-access are the ones that create your account.
Fastest path to seeing something: steps 2 → 4 → 5, then open http://localhost:7701. The prospect site renders with no login and no data.
