## Purpose

Establish the governed model by which sources become retrievable, approved knowledge in the Team Magnificent knowledge library — a library Kevin curates continuously and that agents (esp. Michael, Training Agent / Daily Success Coach) draw on via the Context Manager. This ACR formalizes intake-source classification, the author fast-lane (curation = approval), the reference-only rule for third-party content, and the owned-media persistence path. It conforms to and extends the ratified `KNOWLEDGE_INGESTION_PROTOCOL.md`, `KNOWLEDGE_CORE_RUNTIME.md`, and `KNOWLEDGE_EVOLUTION_RUNTIME.md`.

## Motivation

The ratified knowledge layer defines candidate/review-only ingestion and a Knowledge Evolution runtime that requires an approval reference before knowledge becomes active. Kevin's working reality adds two things the ratified default does not yet name: (1) Kevin authors and curates content directly (training pages, a curated catalogue of THREE's public product videos), and his curation IS the approval; (2) the library must accept mixed media — owned recordings (bytes + transcript) and third-party references (URL + metadata) — which requires a front-door classification and a binary store. Without this ACR, either Kevin's own curated content would sit needlessly in a review queue, or ad-hoc ingestion would drift outside the ratified protocol.

First data source: GitHub `devklg/team-magnificent-training` (`main`) — Kevin's owned training pages (owned-text) + `video-library.html`, a curated catalogue of THREE International's **public YouTube** product videos (reference).

## Scope

### Intake-source classification (front door)

Every source is classified on intake:

- **Owned-text** — Kevin's training pages, notes, ideas → parsed directly into knowledge objects (no hosting, no transcription).
- **Owned-media** — Kevin's recordings/videos → bytes hosted in **Mongo GridFS** (object storage when volume grows); **transcript** generated by **local Whisper (RTX 4070 Ti, GPU)**; transcripts enter as **review-only candidates**.
- **Reference** — third-party hosted content (THREE's public YouTube videos) → **URL + metadata only**, never ingested as stored payload. Optional later enrichment: pull public YouTube captions for richer retrieval.

### Curation policy — author fast-lane

- **Kevin-authored / Kevin-curated content → ACTIVE knowledge directly.** Kevin's authorship is the approval reference `KNOWLEDGE_EVOLUTION_RUNTIME` requires. Fast-lane items are still provenance-stamped and versioned (supersedable/archivable) — curation is the approval, not the absence of a trail.
- **Agent-generated / transcribed / learned / auto-ingested content → REVIEW-ONLY candidates** (ratified default), promoted by Kevin / designated review.

### Knowledge-object model

One canonical Mongo document per item (Mongoose model, per the system persistence standard), with `source_type` variant, content/transcript, product/topic, language, lifecycle/governance status, provenance, and either a reference URL or a GridFS pointer. Embedded in Chroma (GPU, 384-dim) for semantic recall; related in Neo4j (item ↔ product ↔ topic ↔ BA ↔ source).

### Out of scope

- Ratified documents (unchanged; this ACR conforms to and extends them).
- Third-party payload storage (prohibited by the reference rule).
- Agent direct-store access (agents consume via Context Manager only).
- Production code (planning only until approved; implementation is Phase 8).

## Impact

- **Knowledge data-model:** new `source_type` variants and the owned/reference distinction.
- **Ingestion governance:** author fast-lane added alongside the review-gated default.
- **Persistence:** GridFS binary store introduced for owned media (pointer-referenced; not part of the triple-stack).
- **Agent behavior:** unchanged path — retrieval stays Context-Manager-mediated; only the body of approved knowledge grows.
