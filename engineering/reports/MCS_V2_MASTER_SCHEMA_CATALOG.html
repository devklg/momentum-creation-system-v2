0 Identity & membership — the spine

The app is exclusively for Team Magnificent members. A member is an enrolled III International BA in Kevin's downline (III enrollment + downline are hard preconditions; every member IS an enrolled III BA — no member-without-III-BA state). THREE is upstream authority for enrollment/genealogy; the app mirrors the downline slice, never overrides.
  • Member id (canonical): tmagId = TMAG-YYYYMMDD-XXXXXX — the sole login identifier.
  • III mirror (required attribute): threeBaId (+ threeUsername, status) — never authenticates.
  • Sponsor (immutable): captured at signup; must be a member; founders TMAG-01 (Kevin) / TMAG-02 (Paul) are roots.
  • Membership graph: the downline sponsor tree (§7).
Naming — tmag is the Team Magnificent token (decision 2026-07-01): member id tmagId (TMAG-…), access code TMAG-XXXX, and every tm* identifier renames to tmag*. The tables below still show the current pre-migration tm*/TMBA-/TM- names as they exist in code today; the rename of live field names + persisted id values executes in the one governed reidentification migration (§9).
Nomenclature — one concept, one name, cased per layer: both camelCase and snake_case are used, chosen by the software layer (camelCase for TS / app data / Mongo momentum; snake_case for the gateway-memory family, P10 §3.6). A concept has exactly ONE canonical name; casing is a deterministic transform (tmagId ⇄ tmag_id = same concept). Never two different names for the same thing.

2a. Token lifecycle & pool mechanic (the funnel's spine)

Invite-token state machine (invite_tokens.state — forward-only, authoritative; prospects.state mirrors it):

minted → clicked → video_started → video_quarter → video_half
       → video_three_quarter → video_complete → enrolled | expired
  • Pool position anchored at video_complete, monotonic — flushes vacate slots, never renumber (pool_placements.positionNumber).
  • Terminal: enrolled (409) / expired (410). Lazy-flush at read time in /api/p/:token.
  • Separate signals (not token states): callback request (callback_requests) + webinar reservation (webinar_reservations). SSE dashboard updates via poolEvents.
  • becameCustomer is a distinct flag (customer ≠ enrolled BA).
Sponsor immutability: sponsorBaId is stamped when the token is minted and never recomputed. Routes must reject a body sponsorBaId. One exception: Kevin's audited admin override (admin_sponsor_overrides).
