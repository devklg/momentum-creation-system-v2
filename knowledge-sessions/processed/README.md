# Purpose

The **processed** folder contains the machine-generated knowledge extracted from the raw Knowledge Sessions.

These files are **not written by humans**.

They are generated by the Knowledge Ingestion Protocol after processing each Knowledge Session.

The purpose of this folder is to transform human conversations into structured organizational knowledge that can be searched, reasoned over, and used by AI agents.

---

# Philosophy

Raw conversations preserve history.

Processed knowledge creates understanding.

This folder represents the bridge between:

Human knowledge

and

Machine reasoning.

# Context Manager

The Context Manager consumes processed knowledge.

It does not read raw Knowledge Sessions directly.

Instead it retrieves:

Summaries

Decisions

Relationships

Knowledge domains

Implementation notes

Recommendations

and assembles context for agents.

---

# Human Editing

Processed files are **not edited manually**.

If a correction is needed:

1. Update the raw Knowledge Session.

2. Re-run the Knowledge Ingestion Protocol.

3. Regenerate processed knowledge.

The processing pipeline remains reproducible and auditable.

# Source Of Truth

Raw Knowledge Sessions remain the historical source.

Processed Knowledge exists to make those sessions usable by AI systems.

The processed folder is therefore a runtime artifact, not an authoring location.

---

# Final Principle

Momentum learns from conversations.

The raw session preserves the conversation.

The processed session preserves the understanding.

Understanding—not raw text—is what powers the Knowledge Core, Context Manager, and future AI agents.
