# Purpose

The Knowledge Sessions repository preserves the design, architectural, implementation, research, and discovery history of Momentum Creation System V2.

Unlike architecture documents, which define the current source of truth, Knowledge Sessions preserve **how and why** important decisions were made.

They are the research laboratory and institutional memory of the organization.

Knowledge Sessions become one of the primary sources used to grow the organizational Knowledge Core.

---

# Guiding Principle

Architecture tells us what exists.

Knowledge Sessions explain why it exists.

Both are required.

Future AI agents should understand not only the final design but also the reasoning that produced it.

---

# What Is A Knowledge Session?

A Knowledge Session is a structured record of an important design conversation, discovery, architectural breakthrough, implementation discussion, research effort, or strategic decision.

Knowledge Sessions preserve:

* discoveries
* design evolution
* implementation reasoning
* architectural intent
* lessons learned
* future ideas

A Knowledge Session is **not** a governance document.

A Knowledge Session is **not** constitutional authority.

It is an authoritative historical record.

---

# Objectives

Knowledge Sessions exist to:

Preserve organizational learning.

Capture architectural reasoning.

Provide implementation context.

Create organizational memory.

Improve future AI reasoning.

Prevent rediscovering the same ideas.

Accelerate future development.

# Raw

The raw folder contains the original Knowledge Sessions.

These files are never modified.

They are the permanent historical record.

Example:

```text
raw/

KS-0001-....

KS-0002-....

KS-0003-....
```

---

# Processed

The processed folder is created by the Knowledge Ingestion Protocol.

It contains extracted machine-readable knowledge.

Examples:

```text
KS-0001-summary.json

KS-0001-decisions.json

KS-0001-entities.json

KS-0001-relationships.json

KS-0001-keywords.json
```

These files are generated automatically.

Humans do not edit them.
