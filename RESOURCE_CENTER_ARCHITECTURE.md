* Resource philosophy

* Resource lifecycle

Knowledge creates confidence.

# PAGE 1 - GOVERNING PRINCIPLE

## Knowledge Creates Confidence

The Resource Center exists to reduce uncertainty and increase confidence.

It is not a dumping ground for files.

It is not a maze of links.

It is not an archive that only experts understand.

It is a governed knowledge system that helps a person find the right support at the right moment.

Every Resource Center decision must answer:

* Does this resource create clarity?
* Does this resource support action?
* Does this resource strengthen confidence?
* Does this resource align with Team Magnificent values?
* Does this resource preserve compliance and trust?
* Does this resource help a person create momentum?

Knowledge becomes useful when it is discoverable, understandable, current, governed, and connected to action.

---

# PAGE 2 - RESOURCE PHILOSOPHY

## Resources Are Support, Not Clutter

A resource earns its place when it helps a person learn, decide, act, reflect, or support another person.

Resources should be:

* Clear
* Accurate
* Values-aligned
* Easy to find
* Easy to use
* Connected to a journey stage
* Connected to a learning objective
* Governed by ownership

## Resource Center Role

The Resource Center supports:

* Orientation
* Launch Center
* Training
* Daily Success Coach
* CRM
* PMV
* Community
* Events
* Leadership development
* Mentorship
* Agent recommendations

## Resource Boundary

More resources do not automatically create more clarity.

The Resource Center must protect users from overload.

---

# PAGE 3 - KNOWLEDGE PHILOSOPHY

## Knowledge Is Institutional Memory

Institutional memory means the system remembers what the community has learned, approved, refined, and made useful.

Knowledge should not live only in one person's head, one chat, one event, or one outdated file.

## Knowledge Must Be Governed

Every knowledge item should have:

* Source
* Owner
* Version
* Status
* Audience
* Category
* Tags
* Learning objectives
* Review date
* AI usage policy

## Knowledge Is Not Truth By Similarity

Semantic search can retrieve relevant knowledge.

It cannot decide truth.

Truth must resolve back to a governed source record, current version, and approved owner.

Search should make the user feel more capable, not more lost.

Resource search must support:

* Exact lookup
* Category browsing
* Tag filtering
* Semantic retrieval

# PAGE 33 - GRAPHRAG INTEGRATION

## GraphRAG Purpose

GraphRAG combines exact resource records, semantic retrieval, and graph relationships to produce grounded knowledge support.

## GraphRAG Flow

1. Identify the user or agent need.
2. Retrieve exact matching resources from Mongo.
3. Retrieve semantic matches from Chroma.
4. Expand related resources, modules, stages, and principles in Neo4j.
5. Apply governance filters.
6. Produce source-backed answer or recommendation.
7. Record evidence and outcome plan.

## GraphRAG Output

Every Resource Center GraphRAG output must include:

* Answer or recommendation
* Resource IDs
* Source titles
* Version
* Evidence references
* Confidence
* Missing context
* Governance status

## Boundary

GraphRAG may synthesize.

It may not fabricate facts or cite unavailable sources.

# PAGE 40 - RESOURCE CENTER SUCCESS CRITERIA

## Resource Center Succeeds When

* Users find useful resources quickly
* Knowledge reduces uncertainty
* Resources support Orientation, Launch, Training, CRM, Events, and Community
* Michael and agents cite source-backed knowledge
* Training becomes easier to complete
* Daily actions become easier to perform
* Stale knowledge is flagged
* Search results are relevant
* GraphRAG outputs are grounded
* Resource usefulness improves over time

## Resource Center Fails When

* Users feel overwhelmed
* Search returns stale or irrelevant resources
* AI cites unapproved content
* Tags become noisy
* Resources lack ownership
* Chroma is treated as source of truth
* Graph relationships are vague
* Knowledge cannot be audited
* Resources do not connect to action

## Governance Conclusion

The Resource Center is institutional memory.

Knowledge creates confidence.

Confidence creates action.

Action creates momentum.

The Resource Center exists to make the right knowledge available at the right moment, in service of human growth.

People remain the center.
