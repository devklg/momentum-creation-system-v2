# PAGE 1 - GOVERNING PRINCIPLE

## Events Accelerate Momentum

The Event Center exists because community accelerates growth.

Events are not calendar items.

Events are designed moments where people learn together, connect with others, receive recognition, contribute, and feel the culture of Team Magnificent.

Every event must serve at least one constitutional outcome:

* Learning
* Connection
* Recognition
* Collaboration
* Culture reinforcement
* Leadership development
* Momentum creation

An event succeeds when people leave with more clarity, confidence, belonging, or readiness to act.

Events must not become noise.

Events must not create pressure.

Events must create momentum.

---

# PAGE 2 - EVENT PHILOSOPHY

## Events Are Human Infrastructure

The foundation says community is the most important infrastructure.

Events are one of the primary ways that infrastructure becomes visible.

They give members a place to:

* Hear the mission
* Ask questions
* Learn with others
* See leadership modeled
* Receive encouragement
* Experience recognition
* Participate in culture
* Build rhythm

## Events Are Not Content Alone

An event is more than a lesson.

It is a community experience.

Even training events should create belonging and confidence.

## Event Boundary

Events must not be used to pressure, hype, shame, rank, or manipulate.

They must support human growth.

# PAGE 32 - EVENT GRAPHRAG ARCHITECTURE

## GraphRAG Purpose

Event GraphRAG grounds event recommendations and summaries in records, resources, training, community context, and outcomes.

## Retrieval Plan

1. Retrieve event record from Mongo.
2. Retrieve attendance, follow-up, questions, resources, and outcomes.
3. Search Chroma for event summaries, resources, and similar event outcomes.
4. Traverse Neo4j for event-to-member, event-to-resource, event-to-training, event-to-recognition, and event-to-outcome paths.
5. Apply audience and permission filters.
6. Produce grounded recommendation, summary, or follow-up plan.
7. Record evidence and outcome plan.

## Output Requirements

Every event GraphRAG output must include:

* Event ID
* Event type
* Audience fit
* Recommendation or summary
* Evidence references
* Confidence
* Follow-up plan
* Escalation flag

## Boundary

GraphRAG may connect evidence.

It may not invent event impact.
